窃密木马样本分析
发表于 2021-03-05| 更新于 2021-03-06
评论数:

前言

今天在群里看见一个一个师傅分享了一个vbs样本，遂下载下来分析一番

来源 :

  • https://www.virustotal.com/gui/file/8697dc74d7c07583f24488926fc6e117975f8a9f014972073d19a5e62d248ead/details

  • https://s.threatbook.cn/report/file/8697dc74d7c07583f24488926fc6e117975f8a9f014972073d19a5e62d248ead/?env=win7_sp1_enx86_office2013

分析过程-阶段1

打开样本可以看见部分代码和一堆无意义的文本，而且样本经过了混淆，以绕过杀软，很明显可以看到一句POWERSHELL，猜测脚本内嵌入了ps1脚本，通过powershell进行加载

这里的SH 很明显就是WScript.Shell，这段代码会将上面定义的AqUhNbgAqwpMb的内容解密并保存到C:\Users\xxxx\AppData\Local\Temp\OS64Bits.PS1，然后调用powershell执行，中间替换过vbs脚本的路径，猜测是用作持久化

vbscript

分析过程-阶段2

打开OS64Bits.PS1，首先看到的是一串很长的16进制数组，一般都是PE文件，但是这里并不是4D5A开头，估计是压缩了或者加密了

接着往下翻，可以看到一部分不明所以的字符串，而且调用了一个函数Binary2String()，我们可以先将其解码替换了，方便之后的分析

解码之后代码比较清晰了，我们找到脚本的开始位置开始分析

脚本首先复制自身到C:\Users\xxxxx\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup文件夹下以实现开机自启, 之后依次调用VirtualMachineDetector(),DetectSandboxie()检测虚拟机和沙盒环境，这两个函数的原型如下

powershell
powershell

经过以上步骤之后，执行了一行CodeDom $RUNPE "projFUD.PA" "Execute"，估计主要代码应该就在CodeDom函数中，而且之前看到的$RUNPE也被使用了。

经查询，CodeDom是微软提供的源代码生成器，在此用作动态编译，详情：

https://docs.microsoft.com/en-us/dotnet/api/system.codedom

接下来看看这个函数

可以看出，这段代码编译了一段CSharp代码，而且出现了Decompress()，估计之前看到的很长一段代码并不是二进制文件，而是代码，我们将$RUNPE解密保存，这里还使用反射技术加载可疑PE文件

InstallUtil.exe可用作注册系统服务

分析过程-阶段3

我们打开RUNPE.cs，基本可疑看出这是一个将payload载入内存的程序，应该是用作隐藏恶意程序，一番查找后，在github找到了这个:

https://github.com/NYAN-x-CAT/CSharp-RunPE ，这个项目的描述中写道 Hide malware behind a legit process C#

分析过程-阶段4

接下来解密另一个字符数组$Byte, 解压时看见了4D5A，这是个pe文件没毛病，之后用Exeinfo查看，是个.net程序

于是可以使用ILSpy反编译,找到主函数，代码如下：

首先一个Delay3秒，这是过杀软检测的常用方法；

之后可以看到在settings.InitializeSettings()里面做了初始化，进去看看

这里有很多信息被加密了，包含了Hosts和Ports等等，在下方InitializeSettings()中使用aes256进行了解密操作，我们可以直接用他的解密代码，对内容进行解密

Code

这里包含了远程服务器的地址，和一些配置信息，还有一份证书和秘钥，应该是用作和服务器通信的，最后还要通过VerifyHash()验证hash，通过了才能进行下一步

接下来程序创建了一个互斥体用来防止多开，互斥体名就是我们刚刚在settings里面解出来的AsyncMutex_6SI8OkPnk。

然后下一步RunAntiAnalysis()用来分析是否在沙箱检测环境中

DetectManufacturer() 检测vm虚拟机

DetectDebugger() 检测是否被调试

DetectSandboxie() 检测沙盒环境

IsSmallDisk() 检测磁盘大小，一般自动分析环境的磁盘容量不会太大

IsXP() 检测是否在xp系统下

之后，程序检测自身是否已安装，未安装的话进入安装步骤，以实现持久化

这里使用了计划任务, 注册表的方式来实现自启动，根据我们在settings里面看到的InstallFolder = "%AppData%"，可知程序会复制自身到AppData\icrosoft Handler Services下，还有个自删除脚本来启动程序

之后回到主函数，看下一步

这里先判断是否有管理员权限，然后通过RtlSetProcessIsCritical将进程设置为系统严重状态,防止进程被结束

下一步执行PreventSleep(), 设置SetThreadExecutionState函数防止系统挂起，保证程序一直运行

接下来进入死循环，这里开始连接服务器，并将搜集到的信息上传到服务器

这里先判断之前settings里的pastebin是否为’null’，是的话用settings里面的hosts和ports进行连接，否的话应该会从https://pastebin.com/上获取新的主机地址，tcp连接成功，就进入`IL_01b2`，开始信息上传
